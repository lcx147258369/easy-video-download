# 视频抓取与本地下载工具 V1 PRD

## 1. 文档信息
- 文档类型：产品需求文档（PRD）
- 产品阶段：V1
- 面向对象：小范围团队内部使用
- 运行方式：本地 Web 工具

## 2. 产品概述
构建一个本地运行的视频抓取与下载工具。用户输入一个或多个页面 URL，系统自动打开受控浏览器会话，识别页面中的视频资源，并将可下载的视频保存到本地目录。

V1 重点覆盖常见非 DRM 视频页面，优先支持 `mp4`、`webm`、`m3u8`，并支持需要登录态的页面。

## 3. 产品目标
- 让用户从“给出页面 URL”到“视频落地到本地”形成稳定闭环。
- 支持公开页面和需要登录态的页面。
- 支持批量任务队列、任务状态跟踪、失败原因可见。
- 对常见网页视频抓取方式具备较高成功率，包括直链文件、分片流和动态请求。

## 4. 非目标
- 不支持 DRM 受保护内容。
- 不做破解、绕过平台权限或风控的能力。
- 不做云端托管下载，V1 仅在本地机器运行。
- 不做对外账号体系、计费和多人实时协作。

## 5. 目标用户
- 内容运营、研究、归档、采集人员。
- 需要反复从网页中提取视频并本地保存的内部使用者。

## 6. 核心使用场景
- 输入单个视频页 URL，自动识别并下载该页视频。
- 输入多个页面 URL，进入批量队列统一处理。
- 页面需要登录时，用户先在受控浏览器完成登录，再继续抓取。
- 页面包含多个视频资源时，用户可以选择全部或部分下载。

## 7. 核心用户流程
1. 用户打开本地 Web 工具，选择本地下载目录。
2. 用户粘贴一个或多个页面 URL，创建批量任务。
3. 系统为每个任务启动浏览器会话并尝试自动识别视频资源。
4. 若页面要求登录，任务状态变为“需要登录”，用户在受控浏览器完成登录。
5. 系统监听网络请求并识别视频资源，提取真实下载地址及所需请求头。
6. 若识别到多个视频，系统展示候选资源列表，支持全选或部分下载。
7. 系统开始下载，展示进度、速度、文件名和保存位置。
8. 任务完成后，用户可查看历史记录、失败原因并重试。

补充规则：
- 默认走“粘贴 URL 自动处理”主流程。
- 当自动处理失败但页面可交互时，系统允许用户切换到浏览器视图手动触发播放，再继续抓取。
- 登录态按站点域名持久化，后续同域任务复用已登录会话。

## 8. 功能需求

### 8.1 任务输入与队列
- 支持一次输入多个 URL。
- 支持任务状态：`待执行`、`运行中`、`需要登录`、`已识别资源`、`下载中`、`已完成`、`失败`。
- 支持失败任务单独重试。
- 支持按任务查看日志和错误原因。

### 8.2 浏览器会话与登录
- 提供受控浏览器页面，用于加载目标 URL。
- 支持在工具内完成登录流程。
- 登录态按域名保存并复用。
- 当站点需要用户点击播放、切换清晰度或翻页时，允许用户在浏览器视图中手动操作后继续识别。

### 8.3 视频资源识别
- 优先通过浏览器网络监听识别视频请求。
- 兼容直链文件和流媒体地址，首批支持：`mp4`、`webm`、`m3u8`。
- 捕获下载所需上下文：`Cookie`、`Referer`、`User-Agent`、必要请求头。
- 当页面存在多个候选资源时，展示资源列表，包括格式、清晰度线索、大小线索和来源请求。

### 8.4 下载引擎
- 对直链文件执行普通下载。
- 对 `m3u8` 使用流媒体下载方式合并输出本地文件。
- 下载时自动带上识别出的鉴权头和会话信息。
- 支持并发下载，但 V1 默认限制并发数，优先保证稳定性。
- 文件保存到用户指定目录，命名规则默认基于页面标题和时间戳。

### 8.5 历史记录与可观测性
- 保存任务历史、原始 URL、识别结果、输出文件路径和失败原因。
- 提供任务详情页，便于排查站点兼容问题。
- 提供基础统计：成功数、失败数、下载总量。

### 8.6 合规与提示
- 首次使用时展示提示：仅用于抓取和下载用户有权访问和保存的内容。
- 对 DRM 或无法识别的受保护资源明确提示“不支持”。

## 9. 关键产品决策
- 产品形态：本地 Web 工具，不做浏览器插件。
- 范围边界：通用网页视频抓取，排除 DRM。
- 主交互：粘贴 URL 自动处理；必要时可进入浏览器视图补充交互。
- 登录支持：V1 支持完整登录流程，而非仅导入 Cookie。
- 队列能力：V1 直接支持多页面批量任务，而不是只做单任务。

## 10. 验收标准
- 给定一个公开 `mp4` 页面，系统可自动识别并下载成功。
- 给定一个需要登录的普通视频页，用户完成登录后，系统可继续识别并下载成功。
- 给定一个 `m3u8` 页面，系统可正确下载并输出本地可播放文件。
- 给定一个包含多个视频资源的页面，系统可列出候选资源并允许用户选择下载。
- 给定 10 个页面 URL 的批量任务，系统可稳定处理队列，并为每个任务展示最终状态。
- 当页面无法识别视频、资源失效或权限不足时，系统能给出明确失败原因，而不是静默失败。

## 11. 成功指标
- 常见非 DRM 页面的识别成功率达到可用水平。
- 已识别资源的下载成功率达到可用水平。
- 批量任务中，用户可快速定位失败原因并进行重试。
- 团队用户能在不借助开发人员的情况下独立完成日常下载任务。

## 12. 假设与约束
- 用户仅处理自己有权限访问和下载的内容。
- V1 运行在用户本地环境，具备浏览器自动化和本地写文件能力。
- 目标站点以常见网页视频加载方式为主，不包含大规模反自动化对抗。
- V1 先以稳定性优先，性能和大规模并发不作为首要目标。
