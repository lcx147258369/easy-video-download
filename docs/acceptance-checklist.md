# 验收清单

## 构建与静态校验

- [ ] `pnpm typecheck`
- [ ] `pnpm build`

## 本地服务

- [ ] 后端可以在 `http://localhost:4318` 启动
- [ ] 构建完成后，后端根路径可以返回前端页面
- [ ] 若前后端分开跑，前端可以在 `http://localhost:4173` 启动

## 核心业务流

- [ ] 可创建一个或多个抓取任务
- [ ] 任务列表能显示状态
- [ ] 可以打开受控浏览器
- [ ] 页面资源识别成功时，能看到候选视频列表
- [ ] 直链视频可下载到本地
- [ ] `m3u8` 视频可通过 `yt-dlp` 下载
- [ ] 日志面板能看到状态变化或错误信息

## 登录与恢复

- [ ] 需要登录的页面可以先打开浏览器手动登录
- [ ] 登录后可以通过“恢复识别”继续抓取
- [ ] 同站点任务可复用浏览器 profile

## 错误与边界

- [ ] 无视频资源时有明确提示
- [ ] 资源失效时有明确失败原因
- [ ] 不支持 DRM 的提示清晰

## 验证说明

当前项目已经通过的验证重点：

- 共享契约测试
- SQLite 持久化测试
- Browser Manager 持久化 profile 测试
- Video Detector 纯函数与事件收集测试
- Download Manager 单测
- Task Queue 状态流转测试
- API 与 SSE 集成测试
- 前后端构建与类型检查

仍建议补充：

- 用户本机终端中的完整 `pnpm test`
- 真实浏览器手工流程验收
